# ColorService - Material You Theme Colors

## Purpose
Reads `colors.json` generated by matugen and exposes Material You color tokens as QML color properties. All UI components bind to these properties for live theming.

## Architecture
Singleton service. Uses `FileView` to watch the colors.json file and a 2s poll timer to reload it. Parses JSON and converts hex colors to Qt.rgba() values.

## Key Processes / Timers
| Element | Command | Interval | Purpose |
|---|---|---|---|
| `FileView` (colorFile) | Reads `colors.json` | On file change | Triggers `_parseColors()` |
| `Timer` | - | 2000ms | Calls `colorFile.reload()` |

## Key Functions
| Function | Description |
|---|---|
| `_parseColors(jsonText)` | Parses JSON, converts hex → Qt.rgba, skips if unchanged |
| `_hexToR(hex)` | Extracts red component (0-1) from hex string |
| `_hexToG(hex)` | Extracts green component (0-1) from hex string |
| `_hexToB(hex)` | Extracts blue component (0-1) from hex string |

## Color Source
Colors come from `~/.config/quickshell/mrtrotid-shell/colors.json`, generated by `matugen` when wallpaper changes.

## Performance Optimization
`_parseColors()` compares the incoming JSON string with `_lastJson` and skips parsing if unchanged, avoiding unnecessary color property updates.

## Modifying This File
- Add new color token: Add property + JSON key mapping in `_parseColors()`
- Change file path: Modify `FileView.path`
- Change poll interval: Modify Timer interval (default 2000ms)
